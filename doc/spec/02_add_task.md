# 課題追加画面

## 概要

新規課題と初期対策を同時に作成する画面。課題一覧画面からsheet表示される。

## 画面構成

- ナビゲーションバー: 「キャンセル」ボタン、「保存」ボタン
- フォーム:
  - タイトル（TextField、必須）
  - 原因（TextField）
  - グループ選択（GroupSelectorView）
  - 対策（TextField）

## 前提条件

- グループが1つ以上存在する

## 機能仕様

### 入力

| 操作 | 期待結果 |
|------|----------|
| タイトルを入力 | テキストフィールドに入力値が反映される |
| 原因を入力 | テキストフィールドに入力値が反映される |
| グループを選択 | Menuで選択したグループが設定される |
| 対策を入力 | テキストフィールドに入力値が反映される |

### 保存

| 操作 | 期待結果 |
|------|----------|
| タイトル入力済みで「保存」タップ | 課題と対策（入力がある場合）がRealmに保存され、画面が閉じる |
| タイトル未入力で「保存」タップ | 保存ボタンが無効化されているためタップ不可 |

### キャンセル

| 操作 | 期待結果 |
|------|----------|
| 「キャンセル」をタップ | 入力内容を破棄して画面が閉じる |

## 画面遷移

| 遷移元操作 | 遷移先 | 遷移方法 |
|-----------|--------|---------|
| 保存成功 | 課題一覧画面 | sheet dismiss |
| キャンセル | 課題一覧画面 | sheet dismiss |

## バリデーション

| 項目 | ルール |
|------|--------|
| タイトル | 必須。空の場合、保存ボタンが無効化 |
| 原因 | 任意 |
| グループ | 必須（デフォルトで先頭グループが選択済み） |
| 対策 | 任意。入力がある場合のみ対策も同時作成 |

## 補足

- 保存時にFirebaseへバックグラウンド同期される
- グループのデフォルト選択は先頭のグループ
