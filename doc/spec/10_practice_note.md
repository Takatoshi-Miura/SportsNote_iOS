# 練習ノート詳細画面

## 概要

既存の練習ノートを編集する画面。全フィールドがリアルタイムで保存される。

## 画面構成

- ナビゲーションバー: タイトル「練習ノート」、右に削除ボタン（trash）
- Form:
  - 基本情報セクション（BasicInfoSection）: 日付、天気、気温
  - 体調（TextEditorSection）
  - 目的（TextEditorSection）
  - 練習内容（TextEditorSection）
  - 取り組んだ課題（TaskListSection）
  - 反省（TextEditorSection）

## 前提条件

- ノート一覧画面から練習ノート行タップで遷移

## 機能仕様

### 編集

| 操作 | 期待結果 |
|------|----------|
| 基本情報（日付/天気/気温）を変更 | 変更のたびにRealmに即時保存される |
| テキストフィールドを変更 | 変更のたびにRealmに即時保存される |
| 課題の反省テキストを変更 | 変更のたびにRealmに即時保存される |

### 取り組んだ課題セクション

| 操作 | 期待結果 |
|------|----------|
| 「課題を追加」ボタンをタップ | 課題選択ダイアログが表示される |
| 課題選択ダイアログで課題をタップ | 選択した課題が取り組み課題リストに追加される |
| 課題行のメニュー（…）ボタンをタップ | 削除確認アラートが表示される |
| 削除確認で「削除」 | 課題がリストから削除され、関連メモも削除される |
| 各課題の反省テキストを入力 | テキストが即時保存される |
| 未追加課題が0件の場合 | 「課題を追加」ボタンが無効化される |

### 削除

| 操作 | 期待結果 |
|------|----------|
| 削除ボタンをタップ | 削除確認アラートが表示される |
| 確認アラートで「削除」 | ノートが論理削除され、ノート一覧画面に戻る |
| 確認アラートで「キャンセル」 | アラートが閉じ、削除されない |

### ローディング

| 条件 | 表示 |
|------|------|
| データ読み込み中 | ProgressViewとローディングテキストが表示される |
| 読み込み完了 | フォームが表示される |

## 画面遷移

| 遷移元操作 | 遷移先 | 遷移方法 |
|-----------|--------|---------|
| 削除成功 | ノート一覧画面 | dismiss |
| 戻るボタン | ノート一覧画面 | NavigationStack pop |
| 「課題を追加」タップ | 課題選択ダイアログ | sheet |

## 補足

- 全フィールドはonChangeでリアルタイム保存される
- ノート削除時に関連するメモもカスケード論理削除される
- 課題反省テキストはMemoエンティティとして保存される
