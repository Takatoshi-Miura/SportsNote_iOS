# 課題詳細画面

## 概要

既存課題の編集、対策の管理（追加・並び替え・遷移）、課題の完了切り替え・削除を行う画面。

## 画面構成

- ナビゲーションバー: タイトル「課題の詳細」、右に完了ボタン（checkmark.circle）・削除ボタン（trash）
- List:
  - タイトルセクション: TextField
  - 原因セクション: AutoResizingTextEditor
  - グループセクション: GroupSelectorView（Menu）
  - 対策セクション: 対策リスト + 対策追加フィールド

## 前提条件

- 課題一覧画面から課題行タップで遷移

## 機能仕様

### タイトル・原因の編集

| 操作 | 期待結果 |
|------|----------|
| タイトルを変更 | 変更のたびにRealmに即時保存される |
| 原因を変更 | 変更のたびにRealmに即時保存される |

### グループ変更

| 操作 | 期待結果 |
|------|----------|
| グループをMenuで変更 | 変更がRealmに即時保存される |

### 対策リスト

| 操作 | 期待結果 |
|------|----------|
| 対策行をタップ | 対策詳細画面に遷移する |
| 対策が0件の場合 | 「対策なし」メッセージが表示される |

### 対策の追加

| 操作 | 期待結果 |
|------|----------|
| 対策名を入力し＋ボタンをタップ | 対策がRealmに保存され、リストに追加される |
| 対策名が空で＋ボタンをタップ | 何も起こらない |

### 対策の並び替え

| 操作 | 期待結果 |
|------|----------|
| 対策セクションヘッダーの並び替えボタンをタップ | 並び替えモードに切り替わる（ドラッグハンドル表示） |
| 対策行をドラッグ＆ドロップ | 対策の表示順序が変更され、order値が更新される |
| 並び替えボタンを再タップ | 通常モードに戻る |

### 完了切り替え

| 操作 | 期待結果 |
|------|----------|
| 完了ボタンをタップ | 確認アラートが表示される |
| 未完了→完了: 確認アラートで「OK」 | 課題が完了状態になり、課題一覧画面に戻る |
| 完了→未完了: 確認アラートで「OK」 | 課題が未完了状態になり、課題一覧画面に戻る |
| 確認アラートで「キャンセル」 | アラートが閉じ、状態は変わらない |

### 削除

| 操作 | 期待結果 |
|------|----------|
| 削除ボタンをタップ | 削除確認アラートが表示される |
| 確認アラートで「削除」 | 課題が論理削除され、課題一覧画面に戻る |
| 確認アラートで「キャンセル」 | アラートが閉じ、削除されない |

## 画面遷移

| 遷移元操作 | 遷移先 | 遷移方法 |
|-----------|--------|---------|
| 対策行タップ | 対策詳細画面 | NavigationLink |
| 完了切り替え成功 | 課題一覧画面 | dismiss |
| 削除成功 | 課題一覧画面 | dismiss |
| 戻るボタン | 課題一覧画面 | NavigationStack pop |

## 補足

- タイトル・原因・グループの変更はリアルタイムで保存される（onChange）
- 画面離脱時にtaskUpdatedPublisherが発行され、課題一覧画面が更新される
- 削除時に関連する対策・メモもカスケード論理削除される
